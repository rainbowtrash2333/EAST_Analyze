# 银行流水分析系统 

本项目已成功改造为前后端分离架构，提供更好的可维护性和扩展性。

## 项目结构

```

│   ├── api.py                  # Flask API服务器
│   ├── analysis.py             # 数据分析模块

│   ├── network-results.html   # 网络分析结果页

│   │   ├── results.js
```

## 架构特点

### 后端 (Flask API)
- **RESTful API设计**：所有接口返回JSON格式数据
- **CORS支持**：允许跨域请求，支持前后端分离部署
- **健康检查**：提供 `/api/health` 接口检查服务状态
- **文件处理**：支持单文件和多文件上传
- **错误处理**：统一的错误响应格式

### 前端 (纯静态Web应用)
- **响应式设计**：基于Bootstrap 5的现代化界面
- **异步通信**：使用Fetch API与后端交互
- **状态管理**：使用sessionStorage管理分析结果
- **错误处理**：友好的错误提示和状态反馈

## 快速开始

#### 1. 启动后端服务
```bash
cd backend
pip install -r requirements.txt
python api.py
```
后端服务将在 http://localhost:5000 启动

#### 2. 启动前端服务
可以使用任何Web服务器托管前端文件，例如：

**使用Python内置服务器：**
```bash
cd frontend
python -m http.server 8080
```

**使用Node.js http-server：**
```bash
cd frontend
npx http-server -p 8080
```

前端应用将在 http://localhost:8080 可访问

## 功能特色

### 流水分析模式
- 📊 **交易对手分析** - 统计各交易对手的交易次数、金额和收支情况
- 📈 **交易类型统计** - 分析不同交易类型的使用频率和金额分布  
- 📅 **时间趋势分析** - 展示每日和每小时的交易趋势
- 🔍 **多维度统计** - 提供整体统计、渠道分析等多角度洞察
- 📋 **Excel报告** - 自动生成详细的Excel分析报告
- 🖼️ **可视化图表** - 生成多种类型的统计图表

### 网络图分析模式
- 🕸️ **资金流向网络图** - 基于多个Excel文件生成交互式网络图
- 🎯 **智能节点限制** - 自动限制节点数不超过150个，确保图表可读性
- 🔴 **资金流向可视化** - 红色表示资金流出，绿色表示资金流入
- 📊 **网络统计分析** - 提供节点数、连接数、核心账户等统计信息
- 🔬 **深度网络分析** - 度中心性、介数中心性、社区检测、聚类系数分析
- 🏘️ **社区发现** - 自动识别资金流动的社区结构

## API接口

### 基础接口
- `GET /api/health` - 健康检查
- `POST /api/upload` - 单文件流水分析
- `POST /api/upload_network` - 多文件网络分析

### 文件接口  
- `GET /api/download/<filename>` - 下载Excel报告
- `GET /api/charts/<filename>` - 获取分析图表
- `GET /api/networks/<filename>` - 获取网络图

## 文件格式要求

### 流水分析文件格式
Excel文件(.xlsx/.xls)必须包含以下列：

| 列名 | 数据类型 | 说明 | 示例 |
|------|----------|------|------|
| 交易借贷标志 | 文本 | 借/贷标识 | 借、贷 |
| 交易金额 | 数值 | 交易金额 | 10000.00 |
| 对方户名 | 文本 | 交易对手名称 | 张三、某公司 |
| 对方账号 | 文本/数值 | 交易对手账号 | 6226090200001234567 |
| 对方行名 | 文本 | 交易对手银行名称 | 工商银行、建设银行 |
| 现转标志 | 文本 | 现金/转账标识 | 现、转 |
| 交易类型 | 文本 | 交易类型描述 | 网银转账、ATM取现 |
| 交易渠道 | 文本 | 交易渠道 | 网上银行、手机银行 |
| 核心交易日期 | 数值/日期 | 交易日期(YYYYMMDD格式) | 20240101 |
| 核心交易时间 | 数值/文本 | 交易时间(HHMMSS格式) | 143000 |

**数据格式说明：**
- 交易金额：正数，系统会根据借贷标志自动处理正负
- 核心交易日期：8位数字格式(YYYYMMDD)，如：20240315
- 核心交易时间：6位数字格式(HHMMSS)，如：143520表示14:35:20
- 现转标志为"现"时，对方行名将自动设置为"取现"

### 网络图分析文件格式
Excel文件(.xlsx/.xls)必须包含以下列：

| 列名 | 数据类型 | 必需性 | 说明 | 示例 |
|------|----------|--------|------|------|
| 账户名称 | 文本 | 必需 | 主账户名称 | 张三、李四公司 |
| 交易借贷标志 | 文本 | 必需 | 借/贷标识 | 借、贷 |
| 对方户名 | 文本 | 必需 | 交易对手名称 | 王五、某贸易公司 |  
| 交易金额 | 数值 | 必需 | 交易金额 | 50000.00 |
| 证件号码 | 文本 | 可选 | 用于深度网络分析 | 身份证号、统一社会信用代码 |

**数据格式说明：**
- 系统会自动过滤金额小于20万的交易记录
- 为保证网络图可读性，节点数量限制为150个
- 当节点数超过150时，系统自动保留交易金额最大的节点
- 空的对方户名会被自动设置为"取现"
- 只处理借贷标志为"借"或"贷"的有效记录

**网络图方向说明：**
- 借（红色）：资金流出，箭头指向主账户
- 贷（绿色）：资金流入，箭头从主账户指向对方户名

## 技术栈

- **后端**：Python + Flask + pandas + matplotlib + networkx
- **前端**：HTML5 + CSS3 + JavaScript (ES6+) + Bootstrap 5
- **通信**：RESTful API + JSON
- **可视化**：matplotlib (后端) + 动态图表展示 (前端)

## 注意事项

- 支持的文件格式：.xlsx, .xls
- 最大文件大小：16MB
- 分析完成后会自动清理临时文件
- 前后端分离版本需要同时启动两个服务
